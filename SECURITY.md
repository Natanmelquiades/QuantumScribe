# Política de segurança

## Versões suportadas

A versão mais recente publicada na branch `main` recebe correções de segurança.
Versões anteriores podem deixar de receber atualizações.

## Como relatar uma vulnerabilidade

Não publique vulnerabilidades exploráveis em uma issue aberta. Use o recurso
**Report a vulnerability** na aba Security do repositório para enviar um relato
privado, incluindo:

- versão afetada;
- passos mínimos para reprodução;
- impacto observado;
- logs sanitizados, sem transcrições, tokens ou dados pessoais.

O mantenedor buscará confirmar o recebimento em até sete dias. Prazos de correção
dependem da severidade e da complexidade do problema.

## Dados sensíveis

Nunca anexe `config.json`, diários, áudios, modelos ou o conteúdo completo
de `%LOCALAPPDATA%\QuantumScribe`. Remova chaves, prompts pessoais, nomes de arquivos
e transcrições antes de compartilhar evidências.
